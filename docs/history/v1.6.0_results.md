# v1.6.0 实施与验收结果

验收日期：2026-07-18
基线提交：`c6c3337f12935cb5251a966e8ca673366f5e8cc2`（tag `v1.6.0`）
环境：Python 3.11.15、Node v24.15.0

## 已实现

- 独立 `ResearchState`、规则 Router、受控 Structured Planner 和严格 Plan Validator。
- `StepOutputRef`/`ArgumentBinding`、运行前参数解析及工具 Input Schema 二次校验。
- 八个只读 Tool、顺序 Executor、跨 Replan 语义幂等复用和有界工具线程容量。
- Evidence Sufficiency、Direct → Planned 单次升级、最多两次 Replan 和 Partial Answer。
- Context → Draft → Citation Validator → Claim Verifier → deterministic Finalizer。
- 独立 ResearchRunStore（Run、Lease、Plan Version）和独立 SQLite Checkpointer。
- FastAPI lifespan 管理的 Coordinator、lease heartbeat、重启领取、取消和 graceful shutdown。
- 四个始终注册、默认关闭的 Agent API；Idempotency-Key 冲突返回 HTTP 409。
- 30 条版本化 Agent Benchmark、指标实现与离线评测脚本。

v1.4 结构化事实数据库、v1.5 Retrieval 派生索引、旧离线 Analysis Graph 和整个前端均未被
Agent 状态或 checkpoint 替代。

## 安全与隔离

- `langgraph>=1.0.10,<2.0`，`langgraph-checkpoint-sqlite>=3.1.0,<3.2.0`；验收环境实际使用
  LangGraph 1.2.8 和 SQLite Checkpointer 3.1.0。
- Checkpoint pickle fallback 关闭，Msgpack 使用显式项目类型 allowlist；未知类型测试为
  fail-closed，不从 checkpoint 重建成原自定义对象。
- API/Tool 全链路固定 `repo_id + index_version_id`；State 不保存 Secret、Provider、连接、
  handler、整个仓库、完整论文或任意 callable。
- Provider Planner/Answer 只在请求给出 `external_text_consent=true` 且 Provider 可用时执行；
  否则使用本地确定性 fallback。
- 非法 Citation 被删除，模型不能改写路径、行号和页码；unsupported Claim 不保留为最终
  可见确定性结论。

## Feature Flag

`RESEARCH_AGENT_ENABLED=false` 为默认值。关闭时四个路由仍进入 OpenAPI，并稳定返回 HTTP
503 `research_agent_disabled`。启用需要同时安装 `.[agent]` extra；模型和 Dense Retrieval
仍按 v1.5 的独立开关与离线规则运行，自动测试不下载模型、不调用真实 Provider。
